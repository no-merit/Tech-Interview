  <summary><h3>1. 시간복잡도와 공간복잡도에 대해 설명해 주세요.</h3></summary>
  <h4> 알고리즘의 효율성은 알고리즘의 수행 시간 또는 알고리즘이 수행하는 동안 사용되는 메모리 공간의 크기로 나타낼 수 있음.</h4>
<ul>
<li> Big-O, Big-Theta, Big-Omega 에 대해 설명해 주세요.</li>
  입력 크기 n이 무한대로 커질 때의 복잡도를 간단히 표현하기 위해 사용하는 표기법<br>
  Big-O      :최악의 경우, 점근적 상한,n이 증가함에 따라 f(n)이 O(n^2)보다 클 수 없음.<br>
  Big-Theta  :O와 Omega 표기가 같은 경우에 사용. f(n)은 n이 증가함에 따라 n^2과 동일한 증가율을 가진다.<br>
  Big-Omega  :최선의 경우, 점근적 하한,n이 증가함에 따라 f(n)이 O(n^2)보다 작을 수 없음.<br>

  실제로는 Big-Theta로 나타낼 수 있는 표기를 Big-O 표기로 나타냄.

  O(1)의 상수 시간은 입력 크기 n에 대하여 변하지 않고 일정한 시간이 걸림
  
<li> 다른 것을 사용하지 않고, Big-O를 사용하는 이유가 있을까요?</li>

  Big-O는 점근적 상한을 나타내는 표기법으로 해당 알고리즘의 최악의 경우를 나타냄.<br>
  알고리즘의 평균적인 시간은 의미가 없는 경우가 많기 때문<br>
  이정도 시간이 걸릴거야하고 말하는 것보다 절대 이 시간 이상은 넘지 않는다고 말하는 것이 신뢰도가 높음.<br>
  
<li> O(1)은 O(N^2) 보다 무조건적으로 빠른가요?</li>
  복잡도를 계산할 때는 입력 크기 N이 무한대로 향한다고 생각하기 때문에 상수를 무시함<br>
  O(3*N) = O(N), Ω(10logN) = Ω(logN)<br>
  마찬가지로 O(1000)은 O(1)로 표기한다. <br>
  동시에 N<1000 이면, O(1)은 O(N) 보다 느리다.<br>
</ul>

        + 전반적인 시스템의 부하를 줄일 수 있음
        + 단점

          + 입력이 자주 발생하지 않는 컬럼

          + 데이터의 중복도가 낮은 컬럼

    + 원소를 삽입하거나 삭제할 때 원소들을 이동시켜야 해서 시간이 오래 걸릴 수 있음.O(n)의 시간복잡도

  <summary><h3>3. 스택과 큐에 대해서 설명해 주세요.</h3></summary>
<ul>
<li> 스택 2개로 큐를, 큐 2개로 스택을 만드는 방법과, 그 시간복잡도에 대해 설명해 주세요.</li>
  
 + 스택 2개로 큐(fifo의 자료구조)
   + stack1 과 stack2 생성
   + enqueue : stack 1에 원소를 push, O(1) 
   + dequeue : stack 2가 비어있을 경우 stack 1의 원소를 pop 한 뒤 stack2에 push 후 stack2 의 top를 pop ,O(n)
   + dequeue : stack 2가 차있을 경우 stack 2 의 top을 pop, O(1)
 + 큐 2개로 스택을 만드는 방법(lifo의 자료구조)
   + queue1 과 queue2 생성
   + push: queue 1이 비어있으면 queue2에 enqueue,아니면 queue1에 enqueue, O(1)
   + pop : 두 개의 queue 중 비어있지 않은 queue의 원소를 1개만 제외하고 다 다른 queue로 이동한 뒤 마지막 남은 원소를 반환, O(n)
  
<li> 시간복잡도를 유지하면서, 배열로 스택과 큐를 구현할 수 있을까요?</li>
<li> Prefix(전위), Infix(중위), Postfix(후위) 에 대해 설명하고, 이를 스택을 활용해서 계산/하는 방법에 대해 설명해 주세요.</li>

 + 후위표기법
   + 컴파일러가 사용하는 것
   + 피연산자를 먼저 표시하고 연산자를 나중에 표시하는 방법
   + 스택을 사용한 계산 방법 숫자를 만나면 전부 스택에 집어 넣음
     + 연산자를 만나면 스택에서 두 수를 꺼내서 계산하고 다시 스택에 넣음
       + 숫자가 나오면 그대로 출력
       + (*/가 나오면 스택에 push
       + +- 연산이 나오면 (, (가 없다면 스택의 끝까지 출력하고 그 연산자를 스택에 push
       + ) 닫는 괄호가 나오면 (가 나올 때까지 pop
         
<li> Deque는 어떻게 구현할 수 있을까요? </li>

 + deque 덱은 queue의 특성과 stack의 특성을 모두 가지는 자료구조
 + 포인터를 두개 두어서 구현가능
 https://st-lab.tistory.com/183(stranger's lab 자료구조 구현 블로그)

<li> 선형 자료구조</li>
 > 연속적으로 자료 뒤에 자료가 배치되는 구조, 배열, 리스트, 스택, 큐
 
 + 번호(인덱스)와 해당 인덱스에 대응하는 데이터로 이루어진 자료구조
 + 메모리 상에 같은 종류의 데이터들이 순차적으로 저장되고, 인덱스로 해당 자료의 상대적인 위히를 알아 낼 수 있다.
